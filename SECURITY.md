# Security Policy

OSS Security Triage Kit is a defensive tool for open-source maintainers and white-hat researchers.

## Responsible Disclosure

If you believe this repository has a security issue, do not open a public issue with sensitive details. Contact the maintainer privately first and include:

- A concise impact summary
- Affected component or version
- Safe, permissioned reproduction notes
- Evidence that avoids sensitive data, weaponized payloads, or unauthorized access
- Suggested remediation or regression test ideas, if available

## Project Safety Boundary

This project accepts defensive security work:

- Vulnerability report triage
- Pull request risk review
- Remediation planning
- Regression test suggestions
- Responsible disclosure drafting

This project does not support:

- Exploit construction
- Weaponized proof-of-concept material
- Bypass, evasion, persistence, or privilege abuse guidance
- Instructions for unauthorized access
- Requests to target systems without permission

## Maintainer Response Goals

Security reports should be handled with respect for both users and researchers. The goal is to verify risk safely, fix issues quickly, credit responsible researchers when appropriate, and publish clear advisories after users have a reasonable path to patch.
